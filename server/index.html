The Problem

Why this benchmark is genuinely hard.

Real incidents include noisy symptoms, cascading dependencies, and red herrings. A locally plausible action can still hurt terminal score if it misdiagnoses root cause.

A
Typical automation weakness

Overfits recent deployments or obvious noisy services and misses hidden causal chains.

B
What good handling requires

Correlate metrics, logs, config and dependency context before deciding remediation.

C
Why RL fits

Correct close-incident payload quality depends on the sequence of prior investigation actions.

IncidentIQ rewards coherent incident narratives, not keyword matching.
task4_db_connection_limit - common failure mode
# Looks suspicious: analytics-service mem=75% p99=1000ms # Actual root cause: postgres max_connections reduced -> order-service timeouts -> api-gateway 503s # Agent must avoid red-herring fixation.

Reward Design

Dense signal. Delayed truth.

Step rewards encourage useful investigation; terminal reward evaluates final incident report quality.

Investigation
+0.08 root-svc query

Logs/metrics on root service receive stronger reward than generic probing.

Hypothesis
+0.10 correct target

Correct root-cause hypothesis is rewarded before closing the incident.

Terminal
up to +0.60

Close-incidents are scored on root cause, mechanism, remediation, blast radius and efficiency.

Deterministic Grader

Component weight by task.

Every task has explicit weighted components. The table below mirrors the grader logic directly.

Task Root Cause Mechanism Remediation Other Components Efficiency
Task 1: CPU Saturation +0.35 +0.10 +0.25 rollback Blast radius +0.15, no analytics blame +0.15 +0.10 * efficiency
Task 2: Cascading Failure +0.25 +0.20 +0.20 restart No order-service deploy blame +0.15, gateway in blast +0.10 +0.10 * efficiency
Task 3: Silent Corruption +0.20 +0.20 +0.20 config_patch order-service config check +0.15, clean blast +0.15 +0.10 * efficiency
Task 4: DB Connection Limit +0.25 +0.20 +0.20 config_patch postgres config check +0.10, order-service in blast +0.10, clean blast +0.05 +0.10 * efficiency
Task 5: Memory Leak (Twist) +0.25 +0.20 +0.20 config_patch analytics config check +0.10, analytics metrics query +0.10, clean blast +0.05 +0.10 * efficiency
If close_incident is never called, graders give bounded partial credit (capped) from investigation traces.
